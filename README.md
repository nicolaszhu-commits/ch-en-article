# 借鉴中文观点 → 时事驱动的全新英文文章 生成流水线

一个由 Kiro 驱动、可重复运行的工作流：**提炼借鉴**中文观点文章的核心洞见，然后**以英文时事热点为骨架，围绕这个洞见写出一篇全新的英文评论文章**。

## 核心理念（第一性原理）

这**不是翻译，也不是忠实改写**。中文原文只是"思想的种子/透镜"，不是"翻译模板"。

根本标准：成稿与中文原文应**只共享核心观点**，其余（结构、切入点、例子、论证顺序、开头结尾）几乎都不同。一个读过中文原文的人读英文成稿，应觉得"这是另一篇独立的文章，只是碰巧持有相似洞见"，而非"这就是那篇中文的英文版"。

为此：
- 阶段 1 只提炼 1–2 个核心洞见，并**显式丢弃**原文的结构与例子，强制与原文脱钩。
- 阶段 3 **以最新英文新闻为骨架**重新建构文章，洞见只作"透镜"。
- 「事实核查」「文化补桥」是可审计的显式动作；概念译法（内卷/饭圈/直播带货…）可复用。

## 五阶段工作流

1. **提炼核心观点** — 只取 1–2 个可迁移洞见作"种子"，显式丢弃原文骨架
2. **抽象为透镜** — 把洞见抽象成可套用到新事件的普世判断框架
3. **时事驱动选题** — 用 EXA 找最新英文新闻热点作主线骨架，设计原创结构
4. **撰写全新英文文章** — 围绕新闻骨架 + 透镜，原创成文，例子以英文时事为主
5. **润色定稿** — 统一语气节奏，过两道闸门

## 两道质量闸门（贯穿全程）

- **反幻觉**：每一条引入的事实/新闻都必须进入「引用台账」，标注来源、是否已核实、关联强度。未核实或强行拼接的一律剔除或降级。
- **文化补桥**：任何基于中国独有社会现象的论点，抽象后若缺背景，英文读者会觉得逻辑不成立 → 必须显式补充背景解释或类比。

## 终稿自检（阶段 5.1）

终稿后运行零依赖的**原创性/脱钩检测**——确认成稿没有沦为中文原文的翻译版，而是围绕英文新闻独立成文：

```
python3 tools/translation_trace_check.py \
    --source articles/<slug>/source.md \
    --output articles/<slug>/output-en.md
```

在本工作流下追求**低重合**：锚点（数字/专名）顺序重合度应尽量低，≥0.6 即警惕；若共享锚点过多，说明例子没换成英文时事，需补充独立当代案例。最终人工通读判定成稿能否作为一篇独立的、新闻驱动的英文评论成立。

## 前置条件

- 联网搜索：**默认优先使用 EXA**（`web_search_exa` + `web_fetch_exa`），需配置有效 `EXA_API_KEY`。
- 兜底：若 EXA 不可用，自动回退到内置搜索，流水线照常运行。
